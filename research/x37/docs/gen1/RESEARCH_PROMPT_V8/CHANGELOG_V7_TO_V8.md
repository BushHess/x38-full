# CHANGELOG_V7_TO_V8

This changelog lists the material protocol changes made when rewriting V7 into V8.

| Classification | Exact change | Reason |
|---|---|---|
| [FIX] | Added an explicit requirement to report **procedural independence before freeze** separately from **global cross-session split independence**. | V7 allowed those ideas to be inferred, but not stated as separate claims. The V7 execution showed that a procedurally blind run can still be non-independent at the data-split level. |
| [FIX] | Added mandatory freezing and serialization of the **RNG seed(s)**, resample count, and block-size set for every stochastic selection procedure. | The V7 paired-bootstrap protocol was usable, but the original RNG seed was not serialized in the saved artifacts. V8 closes that reproducibility gap. |
| [FIX] | Added a mandatory **common daily UTC return domain** for mixed-timeframe paired tests and paired bootstrap. | V7 required paired comparisons but did not state the common-domain rule directly in the prompt. V8 removes that ambiguity. |
| [FIX] | Added a mandatory **segment trade-count convention** to be locked before scoring and applied consistently across discovery, holdout, reserve, and full-internal reporting. | V7 results were reproducible only after the segment-clipping convention was made explicit in the later reproduction spec. V8 moves that requirement into the protocol itself. |
| [FIX] | Added a rule that the operational definition of a **meaningful paired advantage** must be locked before the first paired comparison is run and must be driven primarily by paired mean daily return on the common daily domain. | V7 used the concept correctly in practice, but the selection logic was not explicit enough in the prompt. V8 makes the comparison standard governable up front. |
| [IMPROVE] | Reworked Stage 1 into a **manifest-first** process: the exact feature-library manifest, parameter ladders, threshold-mode taxonomy, and calibration-mode taxonomy must be frozen before any Stage 1 result is inspected. | This reduces hidden same-file search drift and blocks retroactive library edits after early results are seen. |
| [IMPROVE] | Strengthened Stage 2 with a default cap of **one primary and one backup representative** per closely related family cluster and role unless an explicit discovery-only orthogonality case is written. | V7 already preferred orthogonal representatives, but V8 makes the anti-duplication rule tighter to reduce near-duplicate proliferation and same-file methodological overfitting. |
| [IMPROVE] | Added a hard rule that holdout cannot be used to reopen local parameter neighborhoods, redefine family clusters, or expand the candidate set in any form. | V7 barred new candidates after holdout, but V8 closes softer loopholes that could still reintroduce holdout-driven search drift. |
| [IMPROVE] | Strengthened the reserve/internal governance rule: reserve may corroborate, weaken, or downgrade confidence, but it may **not** replace the frozen winner or reopen the search. | V7 implied this through the freeze stage; V8 makes it explicit because reserve contradiction handling is central to final same-file governance. |
| [IMPROVE] | Kept the **same split dates** as V7 but strengthened the rationale: the split is retained specifically to avoid turning split selection itself into another same-file optimization pass. | After V7, changing the final audit split again would itself become a new search dimension unless a genuine governance failure had been found. None was. |
| [NEW] | Added an explicit requirement that the deliverables include the **serialized stochastic settings used in selection**. | This makes the saved artifacts sufficient for exact procedural reproduction, not just approximate narrative reconstruction. |
| [NEW] | Added an explicit **session-finality statement** to the deliverables and tightened the anti-pattern list to reject any same-file prompt iteration beyond V8. | V8 is positioned as the final same-file convergence audit, not as a stepping stone to V9. |
| [NEW] | Updated the Fresh Re-derivation Rules from **three prior sessions** to **four prior sessions**, and added explicit language that different prior frozen outcomes must not be reconciled by construction. | The session count changed, and the non-privileging rule had to be updated accordingly. |
| [REMOVE] | Removed any wording that could be read as inviting further same-file methodological tightening after V8. | The handoff explicitly positions V8 as the final same-file audit, not as an open-ended prompt-optimization sequence. |
| [REMOVE] | Kept V8 free of session-specific contamination narratives, prior winners, prior feature specifics, and prior parameter regions. | All data-derived specifics belong in the contamination log, not in the clean re-derivation prompt. |
| [IMPROVE] | Tightened the Meta-knowledge section so it contains only **new principle-level lessons** learned from V7 execution. | The handoff required deduplication against V7’s prior lessons and removal of anything that would act as a narrowing prior. |
