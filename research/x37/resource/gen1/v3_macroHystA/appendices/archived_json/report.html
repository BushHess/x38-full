BTC Spot Long-Only — Root-Cause Redesign

This is post-holdout research. The original 2024–2026 holdout was already exposed in prior rounds, so this report is not an untouched-holdout discovery claim. The honest question here is narrower and practical: after seeing where the earlier systems broke, can we identify the real failure mode and redesign the algorithm so the weak regime improves materially without destroying earlier robustness?

Executive conclusion

The earlier thesis was directionally right but mechanically wrong. The durable edge is slow D1 macro trend persistence. H4 should not be treated as the main alpha source or a hot breakout trigger. H4 works best as a state controller with hysteresis: it decides when the macro trend is healthy enough to be in risk, and when that local state has deteriorated enough to step aside.

Root-cause diagnosis

1) The real alpha carrier is the slow D1 trend state, not the fast one

The data does not reward chasing the fastest continuation state. Fast 30–40 day D1 trend states can look excellent in development, but they are less stable across years and much weaker in the later regime. Slower 60-day states give up some headline development Sharpe, but they are materially more stable and hold up much better out of sample.

Interpretation. The right macro variable is not “how hot is the move right now?” but “is the medium-term regime still structurally trending?” That is a slower question.

2) H4 was misused in the earlier systems

Earlier work leaned too hard on H4 as an aggressive continuation trigger. That helped in cleaner trend years but hurt in the later choppier regime. The stronger use of H4 is different: use it to maintain state with hysteresis inside a slow D1 regime.

The important ablation is this: macro only already works, but adding H4 hysteresis is what really repairs late-regime behavior. The optional anti-chase cap improves realized numbers a bit more, but the core gain already appears before that layer, which is why I froze the simpler base system instead of the slightly more optimized extension.

3) The improvement is about staying out of local deterioration, not predicting every breakout

The final system trades more frequently than the macro-only state, but its drawdown is much lower than the fast old-like system and its 2025 trade distribution is much healthier. This is not because it found a magic predictor. It is because it avoids sitting through as much micro-level damage while the slow D1 regime is still technically on.

4) Flow helps, but it is not the root

I tested buyer-imbalance as an entry confirmation layer. It can help a little, but it does not change the story. The main edge is still slow macro trend + H4 state maintenance. I did not freeze the flow filter because its marginal gain does not justify the extra complexity under already-contaminated holdout research.

Final frozen system specification

Name: Slow D1 macro trend + H4 trend-quality hysteresis

Feature formulas:

  • d1_ret_60 = close / close.shift(60) - 1 on D1 bars, then backward-asof aligned to H4 so only completed D1 bars are visible.
  • h4_trendq_84 = (close / close.shift(84) - 1) / (rolling_std(log(close / close.shift(1)), 84, shifted by 1) * sqrt(84)) on H4 bars.

Threshold rules:

  • At each calendar-year start y >= 2020, compute the macro threshold as the 50th percentile of d1_ret_60 over the trailing 1095 calendar days, clipped to start no earlier than 2019-01-01.
  • At each calendar-year start y >= 2020, compute the H4 entry and hold thresholds from all prior H4 bars since 2019-01-01: entry at the 60th percentile, hold at the 50th percentile.
  • 2019 is used as the initial calibration build year. Live trading for this frozen system starts on 2020-01-01.

Trading logic:

  • If flat and d1_ret_60 > macro_threshold and h4_trendq_84 > entry_threshold, enter long at next H4 open.
  • If long and both d1_ret_60 > macro_threshold and h4_trendq_84 > hold_threshold, stay long.
  • If long and either condition fails, exit to cash at next H4 open.

What I believe now

The closest thing to the “truth” in this dataset is not a clever trigger. It is a regime structure:

1. BTC long-only edge lives in medium-term trend persistence on D1.
2. H4 adds value when it controls participation inside that macro regime, not when it chases the hottest breakout.
3. Simpler is safer once the original holdout is contaminated. The base macro+hysteresis system is the right frozen core; extra filters can be layered later, but they are not the source of truth.

Verdict

Best post-holdout redesign so far. It materially improves robustness where the earlier systems were weakest, especially 2025, while preserving strong development performance and acceptable drawdown. I would not call this a final untouched proof, because that would be false. I would call it the strongest practical algorithm discovered so far from the current evidence.
